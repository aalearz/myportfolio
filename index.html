Announcement Center

The Announcement Center is a web-based application that streamlines internal communications by sending formatted announcements, images, and interactive cards to multiple recipient groups via SeaTalk webhook integration. Built with Google Apps Script as the backend API, it features a modern, theme-switchable interface with real-time validation and dynamic content management.

Problem it solves: I developed this during my Shopee internship after observing the manual announcement workflow. Previously, the team had to:

  • Manually send announcements to test groups via Google Script
  • Screenshot the message to show the Team Lead how it looked
  • Wait for verbal/chat approval
  • Manually re-enter and send the same announcement to target groups

My solution: The Announcement Center automates this entire approval hierarchy:

  • User composes the announcement once and selects target groups
  • System automatically sends to test groups first
  • Automatically generates an approval request in SeaTalk, mentioning the Team Lead with an interactive approval button
  • Once approved via the button, the system automatically sends to all selected target groups

Impact: Reduced announcement distribution time by 80% and eliminated manual re-entry, ensuring message consistency across test and production sends.

Asset Issuance Management System

The Asset Management System is a Google Apps Script web application that centralizes and automates employee onboarding, device migration, and asset assignment workflows. It features role-based access control, automated IT PIC assignments, and real-time tracking of device allocations across the organization.

Problem it solves: During my Internship, I identified inefficiencies in the asset management workflow. The team relied on multiple disconnected Google Sheets:

  • HR's onboarding list (accessed via IMPORTRANGE)
  • Asset issuance policies in a separate sheet
  • Device inventory in another document

This fragmented system required switching between multiple sheets, manual data synchronization, and repetitive lookups—making the process slow and error-prone.

My solution: I built a unified web application that integrates three Google Sheets into a single, centralized system:

  • Live HR Integration: Real-time sync with HR's onboarding list
  • Dynamic Policy Management: Editable asset issuance policies directly within the app (add, modify, delete) without touching the source sheets
  • Device Inventory Control: Centralized device tracking with bulk allocation capabilities
  • Automated Workflows: Auto-assigns IT PICs based on predefined rules and generates compliance reports

Technical Implementation: Google Apps Script backend connecting three separate Google Sheets data sources, role-based access control for different team permissions, real-time data validation and policy enforcement, bulk device assignment with automated compliance checks.

Impact: Eliminated the need to navigate multiple spreadsheets, reduced manual data entry, and provided a single source of truth for all asset management operations—improving efficiency and data accuracy across the IT team.

DFA-Based Security Framework for Banking Input Validation

Developed a formal methods-based security system using Deterministic Finite Automaton (DFA) theory to prevent format-based attacks in Philippine banking applications. The system validates PCHC (Philippine Clearing House Corporation) bank codes through a mathematically provable, 6-layer validation pipeline that eliminates format string attacks, SQL injection, null byte injection, and buffer overflow attempts before they reach critical banking infrastructure.

Technical Implementation:

  • Designed and implemented a complete DFA state machine enforcing strict format validation (9-digit codes, "010" prefix, character set constraints)
  • Built a Flask REST API backend with comprehensive attack detection analyzing input patterns in real-time
  • Created an interactive web dashboard with visual state transition tracking and live security monitoring
  • Developed a Windows executable attack simulator for penetration testing and validation accuracy verification

Security Features: Zero false negatives on 100+ attack test cases (100% detection rate), sub-5ms validation time maintaining banking performance requirements, multi-field validation extending to account numbers (13 digits), transaction amounts (decimal precision), and reference codes (alphanumeric), real-time monitoring dashboard displaying attack attempts from external testing tools.

Impact: Achieved 96-100% attack detection rates across format string, SQL injection, XSS, and structural bypass attempts while maintaining zero false positives on legitimate bank codes—demonstrating that formal automata theory provides practical, deployable security solutions for high-stakes financial systems.

Stack: Python, Flask, HTML/CSS/JavaScript (Tailwind), Finite Automata Theory, Security Testing
